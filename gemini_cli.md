
# 프로젝트 소개와 배경에 대한 설명

이 프로젝트의 목적은,  
스프링부트 프로젝트를 생성할 때 마다 ‘환경구성’ 등 매번 반복되는 작업을 줄이기 위해 제대로 된 표준을 만드는거야.  
한마디로 ‘스프링부트 보일러플레이트 프로젝트’ 지.

우선 현재는 버전1이고 목표는 아래와 같아.

- 앱을 구동하는데 필수적인 설정을 표준화 (실무에서 적용해도 될 정도의 품질)
- JPA 개발환경을 통한 데이터 통신
- 간단한 API를 구현하며, ‘요청~응답’ 사이에 필요한 보일러플레이트 구현 (응답문 규격 통일, ExceptionHandler 등) 
- API Doc을 통한 명세 제공, 국제화

이상 버전1의 로드맵이고,
아직 많은 기능에 대처하진 않지만,
애플리케이션의 기본적인 환경 구성에 대한 부분은, 실무에서도 참고 가능할 만큼 표준화 되길 바래.

따라서 여기서 내 질문에 대한 답변을 할때는,
“현재 보일러플레이트라는 목표에 적합한 지”,
“오버엔지니어링이 아닌지”, “실제로 많이 사용되는 기술인지” 에 대해서도 고려를 해줬으면 좋겠어.
